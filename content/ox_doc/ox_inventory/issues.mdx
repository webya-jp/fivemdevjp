# よくある問題

## UIが構築されていない

インベントリの UI は React で記述されているため、FiveM ではネイティブに実行できず、最初に html/css/js にバンドルする必要があります。

バンドル版をダウンロードすることで、これを簡単に行うことができます。[ここ](https://github.com/communityox/ox_inventory/releases/latest)。\
必ずダウンロードしてください`ox_inventory.zip`このファイルにはバンドルされたファイルが含まれており、その他は生のソース コードです。

インベントリ UI を編集したい場合は、これらのファイルを自分で作成する必要があります。\
これを行うには、[*インストール*](../ox_inventory#installation)ガイド。

## リソース ox\_inventory にはそのようなエクスポート \* はありません

この「問題」にはいくつかの原因が考えられます。

* ox\_inventory またはその依存関係の 1 つ (例: ox\_lib) の起動中にエラーが発生しました。
* エクスポートを使用しようとしているリソース（例：esx\_addoninventory）が起動中です*前に*ox\_inventory。
* 文字通り、存在しないエクスポートを呼び出そうとしていますが、これはあなたの問題です。

## スタッシュ/トランクはサーバーの再起動時に保存されません

サーバーを停止したり「再起動」しても、イベントはトリガーされず、保存もできません。

* 在庫は 5 分間隔で保存されます。
* txAdmin のスケジュールされた再起動とシャットダウンにより保存がトリガーされます。
* その`saveinv`コマンドは手動で使用することも、コンソールでトリガーすることもできます。
* オンラインプレイヤーの number が 0 になると、すべてのインベントリが保存されます。

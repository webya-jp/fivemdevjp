# サーバーリブ

## スポーンビークル

サーバーサイドで車両を作成し、そのネットIDを返す。これがプレイヤー所有の車両で、persistence convarが有効になっている場合、スポーンされた車両はpersistされる。

```lua
qbx.spawnVehicle(params)
```

* params：`table`
  * モデルである：`integer`
    * 車両モデルのハッシュ。
  * スポーンソース`integer | vector3 | vector4`
    * ビークルをスポーンするエンティティまたは座標のハンドル。
  * ワープ？`boolean | integer`
    * スポーン後に車両にワープするためのオプションのペッドID。
    * もし`spawnSource`を渡すことができます。`true`でワープさせる。
  * 小道具？`table`
    * ox\_libの[車両特性](https://coxdocs.dev/ox_lib/Modules/VehicleProperties/Client#vehicle-properties).
  * バケツ？`integer`
    * スポーンされたエンティティを移動させるオプションのルーティングバケット。

リターンズ

* netId：`integer`
* 実体を持つ：`integer`

```lua
-- spawns the vehicle at `myVectorCoords` and warps `myPed` inside
qbx.spawnVehicle({
  model = `asbo`,
  spawnSource = myVectorCoords,
  warp = myPed,
})

-- spawns the vehicle at `myPed` and warps the same ped inside
qbx.spawnVehicle({
  model = `asbo`,
  spawnSource = myPed,
  warp = true, -- causes `myPed` to be warped into the vehicle
})
```
